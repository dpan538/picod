# Picod Shrine + Torii Specific Shape Rules

Version: 0.1
Scope: `shrineSmall`, `torii`
Purpose: ensure ceremonial identity reads first and both assets belong to the same ritual system.

## 1) Ceremonial Identity Rule
- `shrineSmall` and `torii` must read as ceremonial first, architectural second.
- Centerline clarity is prioritized over decorative density.
- Threshold readability must survive map zoom.

## 2) Shrine Silhouette Rule
- Strong frontal orientation.
- Roof wider than doorway mass, but not toy-like oversized.
- Clear central threshold signal (opening/recess) is mandatory.
- Lower body mass must remain readable.
- Near-axis symmetry is mandatory.

Avoid:
- Hut/kiosk-like square reads.
- Body too narrow under roof.
- Side-heavy asymmetry.
- Residential window-door pairing language.

## 3) Shrine Front Rule
- Front should read as ritual threshold, not domestic entrance.
- Prefer centered opening + framed lower posts.
- Keep center clean; avoid noisy checker details.
- Red accents allowed only if sparse and intentional.

## 4) Shrine Roof Rule
- Calm, stable roofline with ceremonial spread.
- Roof underside darker than top plane.
- Keep roof/body balance so body does not disappear.

Avoid:
- Tiny peaked cap.
- Over-steep toy roof.
- Full-edge decorative noise.

## 5) Torii Shape Rule
- Must read instantly as torii, not generic red gate.
- Two clear posts + one dominant upper lintel.
- Secondary beam optional only if zoom readability remains.
- Torii should be visually lighter than shrine body mass.

Avoid:
- Doorframe proportions.
- Roof-like heavy top beam.
- Building-like lower mass.

## 6) Shrine–Torii Relationship Rule
- Same ceremonial axis, different density roles.
- Torii: lighter and more symbolic.
- Shrine: denser and more enclosed.
- Red is stronger on torii; shrine uses red sparingly.

## 7) Grounding Rule
- Follows `PICOD_DEPTH_CUE_POLICY.md`.
- Grounding for ceremonial assets should be subtle and centered.
- No raised-platform illusion, no side-wall illusion.

## 8) Acceptance Checklist
- Shrine reads as shrine before generic small building.
- Torii reads as torii within <= 1 second at map zoom.
- Pair feels related but not redundant.
- Shrine centerline remains clean.
- Torii does not gain building-like mass.
- Neither asset drifts into toy-like proportions.
